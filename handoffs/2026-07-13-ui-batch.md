## Decisions (for review) — why each was made

1. **R2 scoring — "raise the cap," not exponential.** The complaint ("we lose info on large errors") was about the `max(0, 30−E)` **floor**, not the L1 distance. Fix = keep `max(0, base − E)` but raise `base` so the floor almost never binds (only troll bets E>100/150 floor). Chose this over exponential decay (option 3, needs engine surgery + slight incentive departure) and over "no floor / can go negative" (option 2). **Config-only** — `scoring.py` `bet_score` unchanged; constants live in `app_settings.scoring_constants` = `{poll:{base:100,per_seat:1}, final:{base:150,per_seat:1}}`. docs/02 §3/§4 + `test_scoring.py` updated. **Frozen at launch.**
2. **R3 — final worth 1.5× polls (not ~3×).** Final cap 150 = 1.5× poll cap 100; both scored once/week ⇒ ~1.5× aggregate.
3. **R4 — mergers removed from player copy only.** Lists are final/official at publish, so no mid-game mergers expected. The common-partition **engine is kept dormant** (still covered by `test_scoring.py`) as insurance. docs/02 §6 notes this.
4. **R5/R6 — tie-breakers + fair-play copy removed.** Tie-break order still lives in docs/02 §5 as internal spec, just not shown to players.
5. **R7 — Friday→Friday, fieldwork-date basis.** Owner asked for "polls published Friday to Friday" and initially chose *publication date*. **Discovered Wikipedia exposes no publication date** (inspected `pipeline/fixtures/wikipedia_2026-07-11.json` + `scraper.py` META_COLUMNS) — only fieldwork dates. Owner then chose **fieldwork_end** basis. Week model = **Friday 00:00 → Thursday 23:59**, lock at the week's Friday noon. A nullable `publication_date` column was added for the day a real source exists. **"published" in the rules is colloquial — internally it's the fieldwork date.** Anti-sniping argument re-derived in docs/02 §7 (residual one-poll edge <1 seat on the unweighted mean).
